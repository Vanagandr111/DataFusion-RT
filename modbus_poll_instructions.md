# Инструкция по тестированию с Modbus Poll

## Установка
1. Запустите `driver\ModbusPollSetup64Bit.exe`
2. Следуйте инструкциям установщика
3. После установки запустите Modbus Poll

## Настройка соединения

1. **Connection → Connect...** (или F3)
2. В диалоге настроек:
   - **Connection:** Serial Port
   - **Serial Port:** COM9
   - **Baud Rate:** 9600
   - **Data Bits:** 7
   - **Parity:** Even
   - **Stop Bits:** 1
   - **Mode:** RTU
3. Нажмите **OK**

## Настройка запросов

1. **Setup → Read/Write Definition...** (или F8)
2. В диалоге:
   - **Slave ID:** 1
   - **Function:** 03 Read Holding Registers
   - **Address:** 6 (0x0006)
   - **Quantity:** 3
   - Нажмите **OK**
3. Добавьте дополнительные запросы:
   - Функция 03, адрес 21 (0x0015), количество 3
   - Функция 03, адрес 86 (0x0056), количество 3
   - Функция 04, адрес 90 (0x005A), количество 3

## Интерпретация результатов

- **Зеленые значения:** Успешный ответ от устройства
- **Красные значения:** Ошибка или таймаут
- **Статистика:** В нижней части окна отображаются счетчики успешных/неудачных запросов

## Дополнительные настройки

1. **Setup → Communication...**
   - **Response Timeout:** Увеличьте до 2000 мс
   - **Delay between polls:** Установите 500 мс
2. **View → Communication Traffic** - для просмотра raw трафика

## Тестовые сценарии

### Сценарий 1: Проверка связи
- Настроить один запрос (адрес 6, функция 03)
- Если получаете ответ → адаптер работает
- Если нет ответа → проблема в адаптере/подключении

### Сценарий 2: Сравнение с MCGS
- Подключить MCGS обратно к линии
- Запустить Modbus Poll в режиме прослушивания (только прием)
- Проверить, видит ли Modbus Poll трафик MCGS

### Сценарий 3: Тест управления направлением
- В настройках соединения попробовать разные опции RTS:
  - RTS Control: Disabled
  - RTS Control: Enable (возможно RTS/CTS)
  - RTS Control: Toggle (наиболее вероятно для RS-485)

## Сохранение конфигурации
- **File → Save** для сохранения настроек
- Можно сохранить как `modbus_test.mp` для повторного использования

## Устранение неполадок

### Нет ответа от устройства
1. Проверьте физическое подключение (A/B линии)
2. Убедитесь, что печь включена
3. Проверьте Slave ID (обычно 1)
4. Попробуйте другие адреса регистров (0, 1, 2)

### Ошибки CRC
1. Проверьте настройки четности (должна быть Even)
2. Проверьте количество стоп-битов (должно быть 1)
3. Проверьте скорость (должна быть 9600)

### Прерывистая связь
1. Увеличьте Response Timeout
2. Проверьте наличие терминаторов (120 Ом на концах линии)
3. Проверьте качество соединений
