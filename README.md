# DataFusion RT

Windows GUI-приложение на Python для сбора данных с лабораторных весов Adam Highland HCB и печи с PID-контроллером по RS-485 / Modbus RTU.

## Что умеет текущая версия

- GUI на `tkinter` со встроенным live-графиком `matplotlib`;
- старт окна в развернутом режиме, адаптивная раскладка под разные экраны;
- светлая и тёмная темы с сохранением выбора в `config.yaml`;
- сканирование COM-портов с описанием устройств;
- назначение COM-порта для весов и печи прямо из GUI с записью в YAML;
- проверка весов и печи отдельными кнопками;
- крупная приборная зона под графиком: масса, температура, статус, `SV`, timestamp, число сэмплов;
- экспорт накопленных данных в `CSV`, `XLSX` и `XML`;
- `test_mode` для проверки интерфейса и графика без оборудования;
- логирование в консоль, GUI и `logs/app.log`.

## Установка

```bat
scripts\install.bat
```

Если окружение уже создано:

```bat
.venv\Scripts\activate.bat
python -m pip install -r requirements.txt
```

## Запуск GUI

```bat
python app\main.py --config config\config.yaml
```

или:

```bat
scripts\start.bat
```

По умолчанию приложение стартует максимально развернутым. Если в конфиге включить `fullscreen: true`, окно откроется в fullscreen. Выход из fullscreen: `Esc`.

## Работа с интерфейсом

Слева:

- список COM-портов с описанием;
- кнопка `Обновить порты`;
- назначение выбранного порта на весы или печь;
- проверка устройств кнопками `Проверить весы` и `Проверить печь`;
- управление запуском, остановкой, сбросом, тарой, нулём и экспортом.

Справа:

- live-график с двумя осями Y:
  - масса;
  - температура;
- большая нижняя панель текущих значений;
- живой лог.

## Темы

Поддерживаются 2 темы:

- `dark`
- `light`

Тема выбирается в верхней панели GUI и сразу сохраняется в `config/config.yaml`.

Важно: график оформлен отдельно от UI и остаётся светлым “научным” даже в тёмной теме интерфейса, чтобы не терять читаемость.

## Экспорт данных для Excel

Кнопка `Экспорт данных` открывает стандартный диалог сохранения файла.

Поддерживаются форматы:

- `CSV` — основной формат для Excel;
- `XLSX` — прямой экспорт Excel;
- `XML` — дополнительный вариант.

Экспортируются поля:

- `sample_index`
- `timestamp`
- `mass`
- `furnace_pv`
- `furnace_sv`

Если данных ещё нет, приложение не падает и показывает понятное сообщение.

## Test Mode

Если включить `test_mode`, приложение работает без оборудования:

- масса плавно уменьшается;
- температура плавно растёт;
- график, лог и экспорт можно проверить дома.

Включить можно:

- галочкой `Test mode` в GUI;
- или через `app.test_mode: true` в `config/config.yaml`.

## Конфиг

Основной файл: `config/config.yaml`

Минимальный актуальный пример:

```yaml
scale:
  enabled: true
  port: COM3
  baudrate: 4800
  timeout: 1.0
  mode: auto
  request_command: "P\r\n"

furnace:
  enabled: true
  port: COM4
  baudrate: 9600
  bytesize: 8
  parity: "N"
  stopbits: 1
  timeout: 1.0
  slave_id: 1
  register_pv: 0
  register_sv: 1
  scale_factor: 0.1

app:
  poll_interval_sec: 1.0
  csv_path: "data/measurements.csv"
  log_path: "logs/app.log"
  max_points_on_plot: 500
  test_mode: false
  theme: "dark"
  start_maximized: true
  fullscreen: false
  default_export_format: "csv"
```

### Что важно в `app`

- `theme`: `dark` или `light`
- `start_maximized`: стартовать развернутым
- `fullscreen`: открыть в fullscreen
- `default_export_format`: формат по умолчанию для экспорта

Если этих полей нет в старом конфиге, программа подставит разумные значения по умолчанию.

## COM-порты

GUI показывает порт в формате:

- `COM5 - USB-SERIAL CH340`
- `COM7 - Silicon Labs CP210x`
- `COM3 - Bluetooth`

Выбранные порты сохраняются обратно в YAML:

- `scale.port`
- `furnace.port`

Даже если устройство сейчас не подключено, порт можно вписать вручную и сохранить.

## Сборка EXE

scripts\build_exe.bat

## Типовые проблемы

### COM-порт не найден

- нажмите `Обновить порты`;
- проверьте драйвер USB-Serial / USB-RS485 адаптера;
- убедитесь, что порт не занят другой программой.

### Весы молчат

- проверьте режим передачи на весах;
- проверьте `4800 8N1`;
- попробуйте `scale.mode: poll` или `auto`;
- убедитесь, что выбран именно USB COM, а не Bluetooth.

### PID не отвечает по Modbus

- проверьте `slave_id`;
- проверьте `baudrate`, `parity`, `stopbits`;
- проверьте адреса `register_pv` и `register_sv`;
- учтите, что реальные регистры контроллера нужно подтвердить на стенде.
